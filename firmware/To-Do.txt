1. Device: Sende Datensatz zum Server (Alive-Datensatz senden)

2. OEM-Server: Empfängt Firmware Datensatz vom Device

3. OEM-Server: Abfragen von Hardwareinformationen vom Device

4. Device: Sammeln und Bereitstellen von Hardwareinformationen
- df_kvp=exinfo; df_kvp=exinfo,ac,021,1 (...ac,<Modul>,<Master>)

5. OEM-Server: Firmwareversion checken (OEM-Server entscheided wie die Firmware deployed wird)
- 1 OEM fetches firmware-IFF from DFZ file
- 2 OEM delivers firmware as IFF response

6. Device: Download-Anfrage erhalten, Download auslösen
- df_load_firmware=<DFZ-File>,<FW-File>
- lade Firmware update file von Firmwareupdate-Endpoint

7. Fileserver: Sende IFF content zum Device

8. Device: Empfange Daten vom Fileserver

9. Device: Auswerten der Firmwarekompatibilität 
- Incompatible -> Systemmeldung an OEM-Server <Incompatible>
- Compatible -> Schritt 10

10. Device: Anwenden von Firmware auf das Gerät
- Setup und Listen behalten

11. Device: Neustarten

12. Device: Systemmeldung an OEM-Server <Installation Success>


Ermitteln der neusten Firmware-Version:
127.0.0.1/api/firmware/query.php?fw=latest
